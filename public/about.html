Background

Domain-specific modeling

Domain-specific modeling (DSM) makes software more accessible and inclusive by leveraging the expertise and knowledge of domain experts from various backgrounds instead of relying solely on programmers and software engineers. This modeling practice allows the solution to be specified directly using problem domain concepts.

Language workbenches

Due to their specific nature, common programming tools (perfected throughout the years), like editors E.g. VS Code, Sublime Text and IDE E.g. Eclipse, Visual Studio are inadequate for DSLs used by domain experts of diverse backgrounds. Language workbenches were created to support these modeling and language engineering activities
Ref.
  • Web Language workbenches: The killer-app for domain specific languages (2005)
    • Fowler, Martin
They support the efficient definition, reuse, and composition of languages and their IDEs. Most language workbenches use a parser-based approach E.g. Xtext and MetaEdit+ where an abstract syntax tree (AST) is progressively built by scanning the input and validated against a grammar. Since various experts may use a model with different needs, it should be possible to present the model in different forms, equivalent to one another. This is found in projectional editing, E.g. Jetbrains MPS, Scratch, which allows the AST to be modified directly; thus enabling support for multiple notations easily interchangeable and composable.
Although promising as an approach, current solutions are heavy-weight, cannot be easily integrated with other tools, and suffer from poor usability. Additionally, many language workbenches rely on the Unified Modeling Language (UML), which is rooted in object-oriented programming (OOP). Therefore, they hinder the modeling process and restrict the expression of a concept to the scope and principles of OOP.

Gentleman

To better support this paradigm, Gentleman was initiated with the aim to bridge the gap between model and domain expert. It is a web-based modeling editor that offers a user-friendly environment that is easily integrable through projectional editing. With Gentleman, models are created by combining concepts (data structures and constraint rules) and projections (visuals and interactions). In our approach, the domain concepts are defined with structures and languages unrelated to any programming paradigm, considering modeling as an activity of its own. Gentleman is offered as a JS library that can easily be integrated into a larger web application.
